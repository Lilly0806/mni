Descrição:
Tipo de elemento que encapsula os dados de consulta relativos à existência de modificações em um processo judicial. Só é possível consultar a alteração se uma das partes do processo estiver associada a um procurador, se não, o sistema apresenta o erro “Processo não encontrado”.

Dados criminais:
Os dados criminais serão incorporados de forma estruturada, os campos terão os detalhamentos do delito do polo passivo (parte processada), para cada pessoa no polo passivo deve ter, o nome e o numeroDocumentoPrincipal. O Objeto delito com a dataDelito, localDelito, ufLocalDelito. E por fim, a lista de enquadramentos listaEnquadramentos com o código do crime ou dispositivo legal em idNormaDispositivo e se o crime foi tentado crimeTentado, com respostas S para sim e N para não.

Regras para preenchimento do bloco

• O nome do réu deve ser informado em caixa alta.

• O campo localDelito corresponde ao código IBGE da localidade do fato.

• O elemento idNormaDispositivo representa o enquadramento penal e deve ser obtido na tabela de apoio NormaDispositivo, disponível nos serviços abaixo:
